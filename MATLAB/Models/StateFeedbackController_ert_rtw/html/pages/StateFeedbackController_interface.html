1. はじめに [show]

コード ジェネレーターによりモデルのデータ コード インターフェイスが生成されました。コード インターフェイス レポートには、モデルを呼び出し可能なエントリポイント関数、およびルートレベルの入出力端子などのデータ インターフェイス要素の概要が示されています。

2. 関数インターフェイス [hide]

外部コードは、モデルに対して生成されたエントリポイント関数を呼び出して、生成コードの実行を開始します。コード ジェネレーターがモデルに対して生成する関数コードのタイプは、モデルで表現されるモデリング スタイルと周期的レートに応じて異なります。詳細については、「生成された C 関数インターフェイスをモデルのエントリポイント関数用に構成」を参照してください

2.1 初期化関数

初期化エントリポイント関数は起動動作を実装します。モデルで、Initialize Function ブロックは起動動作を明示的に表現します。

StateFeedbackController_initialize

生成コードの初期化のエントリ ポイント
#include "StateFeedbackController.h" void StateFeedbackController_initialize(void)

2.2 終了関数

終了エントリポイント関数はシャットダウン動作を実装します。モデルで、Terminate Function ブロックはシャットダウン動作を明示的に表現します。

モデルで終了関数は表現されていません。

2.3 周期関数

周期的なエントリポイント関数は、固定サンプル レートで発生するモデルの動作を実装します。レートベース モデルについて、コード ジェネレーターはモデルで使用されるレートごとに周期的なエントリポイント関数を生成します。関数呼び出し (エクスポート関数) モデルで、サンプル レートを指定する Function Call Subsystem ブロックは周期関数を表現します。

StateFeedbackController_step

生成コードの出力のエントリ ポイント. 0.001 秒ごとに定期的に呼び出されなければなりません.
#include "StateFeedbackController.h" real_T StateFeedbackController_step(real_T arg_ref_x, real_T arg_x, real_T arg_theta, real_T arg_gain_k[4])

2.4 非周期関数

非周期的 (非同期) なエントリポイント関数は、非同期で発生するコンポーネントの動作を実装します。関数呼び出し (エクスポート関数) モデルで、サンプル レートを指定しない Function Call Subsystem ブロックは非周期関数を表現します。

モデルで非周期関数は表現されていません。

3. データ インターフェイス [hide]

データ コード インターフェイスは、ルートレベルの Inport や Outport などのモデル データ インターフェイス要素をストレージ クラスにマッピングします。ストレージ クラスは、生成コード内でのデータ要素の外観と配置を指定します。ストレージ クラスは直接アクセス データ通信を適用します。マッピングに使用可能なストレージ クラスのリストは、モデルに関連付けられている Embedded Coder ディクショナリで決まります。モデル コード マッピングでは、モデル インターフェイス要素をストレージ クラスにマッピングします。

3.1 入力端子

入力は、モデルのルートレベルの入力端子に対するストレージ クラスの指定に基づいてコード内で生成された変数です。

arg_ref_x

<Root>/ref x の入力データ
外部で定義
real_T arg_ref_x;

arg_x

<Root>/x の入力データ
外部で定義
real_T arg_x;

arg_theta

<Root>/theta の入力データ
外部で定義
real_T arg_theta;

arg_gain_k

<Root>/GainK の入力データ
外部で定義
real_T arg_gain_k[4];

3.2 出力端子

出力は、モデルのルートレベルの出力端子に対するストレージ クラスの指定に基づいてコード内で生成された変数です。

arg_out

<Root>/out の出力データ
外部で定義
real_T arg_out;

3.3 パラメーター

パラメーターは、モデル パラメーターおよびモデル パラメーター引数を表す、生成コード内の変数です。変数は、ストレージ クラスの指定に基づいて生成され、ターゲット環境でのモデル コード実行の際に調整可能かそうでないかのフラグが設定されます。

モデルにインターフェイス パラメーターまたは調整可能なパラメーターがありません。

3.4 内部データ

内部データは、信号データ、内部状態をもつブロック、およびデータ ストアを表す、生成コード内の変数です。変数は、ストレージ クラスの指定に基づいて生成され、ターゲット環境でのモデル コード実行の際に測定するかどうかのフラグが設定されます。

グローバル データ ストアまたは非自動ストレージ クラスのデータ ストアがモデルにありません。
